Bomberman Pokémon : PokéBomber

Projet : Bomberman Pokémon : PokéBomber

Création d'un jeu clone de Bomberman en C# accessible depuis son terminal et avec support du multijoueur en réseau. Ce qui rends ce projet unique, c'est qu'il reprends le thème des jeux Pokémon 2B Topdown.

Démonstration du Projet

Voici les instructions à suivre pour pouvoir tester le projet vous-même :

Lien du Github avec les fichiers nécessaires pour l'installation.


Pré-requis


- Avoir une vm graphique (Linux, debian)
- Avoir installé gcc (compilateur langage C) :

apt-get install gcc

Pour installer toutes les librairies utiles :

sudo apt install build-essential

- Savoir jouer

Installation


Excuté le script dependance.sh,

Démarrage


Pour lancer le jeu :
make
./Bomberman

une instance du jeu va s'ouvrir sur votre VM

Bonus 1/3 : Le Menu principal

Depuis cette page, le joueur peut choisir de jouer seul contre l'ordinateur, en multijoueur local sur un même clavier, ou en multijoueur en ligne avec d'autres joueurs ayant installé le jeu sur leur terminal. "Exit" redirige vers l'écran Start.

Bonus 2/3 : Le menu "Options"

Depuis cette page, le joueur peut consulter les touches attribuées au joueur 1 et 2 (les joueurs local jouent sur le même clavier), peut couper le son du jeu et le réactiver, et revenir au Menu principal.

Bonus 3/3 : Background utilisé pour la Map du jeu

Il s'agit du fond décoratif de la Map.
À la différence de la Map sur laquelle les joueurs sont placés, restrients, et peuvent intéragir, le Background décore le reste de la zone non-accessible.
Chaque sprite a été designé à la main ou repris de designs existants, et adaptés par mes soins pour respecter la taille de 34x34px pour rester en accord avec les blocs qui créent la Map.

Bonus 3/3 : Map utilisée pour définir la zone jouable

Il s'agit de la Map du jeu.
À la différence du Background décoratif qui sert à donner vie au jeu, la Map définie simplement les limites de l'espace jouable auquel les joueurs peuvent accéder.
Concernant le sprite utilisé pour les bordures, l'arbre est également 34px de large, et il a été décidé que la Map serait fixe entre chaque partie, bien qu'ajouter une fonctionnalité de choix de forme de carte aléatoire serait envisageable.
